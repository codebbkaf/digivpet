CARRIED-FORWARD FACTS worth knowing before iteration 1 — the full detail is in the
archived progress.txt, but these three cost previous iterations real time:

1. DO NOT UNINSTALL THE APP to reset game state. It wipes the HealthKit grant and
   every launch then sticks on the system health sheet, which no simctl command can
   tap. Repair means editing `authorization` in
   ~/Library/Developer/CoreSimulator/Devices/<UDID>/data/Library/Health/healthdb.sqlite
   (status 103 == granted). Add a #if DEBUG launch flag instead. US-059, US-068 and
   US-069 are the stories most likely to reach for an uninstall.

2. EvolutionGraph.bundled fatalErrors on a decode failure, and the app is the
   TEST_HOST — so one JSON typo kills the whole suite before a single test reports,
   and xcodebuild blames "test runner crashed before establishing connection". The
   real diagnostic is above it: grep for "Could not load the evolution graph".
   US-056 and US-061 both edit that file.

3. simctl cannot synthesise a tap. Any screen that needs a screenshot must be
   pushable from a #if DEBUG launch argument — the established pattern is
   -dexLineDemo / -dexDetailDemo / -complicationDemo / -wanderDemo.

Sources/Complication/RefreshGranularitySpike.swift is DELIBERATELY still present —
it is the instrument for the AOD repaint measurement, which remains the one open
