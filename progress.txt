## 2026-07-22 — US-110: Feeding, training or battling wakes a sleeping Digimon
- The shape of the fix is ONE new private method, `MainScreenModel.wakeIfAsleep()`, called at the top
  of `feed()`, `train()` and `battle()` where `noteWakingEarly()` used to be called at the bottom of
  their blocked arms. `grep -rn noteWakingEarly Sources/ Tests/` now returns nothing. It does three
  things and all three are load-bearing: `recordWakingEarly` (untouched — every disturbance counts in
  `stageSleepDisturbances`, at most one a day in `careMistakeCount`), stamps `state.awakeUntil`, and
  clears the model's `isAsleep` there and then, because the actions read it in the same call stack and
  the next re-derivation is a whole refresh away.
- AC8 is the one that would have shipped this broken. `updateSleepState()` used to ask
  `sleepSchedule.contains(now())`, which every foreground would have used to undo a wake the user had
  just been charged a care mistake for. It now asks `SleepSchedule.isAsleep(at:wokenUntil:)` — a new
  pure method beside `contains`, so the override lives in ONE place. The two DEBUG demo forcers were
  moved onto it too, or `-sleepDemo` would have been the third copy.
- The marker is `GameState.awakeUntil: Date?`, an optional storage property + computed accessor like
  every other one there (the SwiftData migration shape). AC9 needs no sweeper: it is an ABSOLUTE
  instant, so once past it reads expired forever and last night's 02:05 cannot make tonight's 02:00
  awake. There is one exception — `forceAsleepForDemo` clears it, because the Simulator's store
  survives between launches and a `-sleepDemo` run within five minutes of the last one would have
  opened on a Digimon still awake from that run's Feed tap.
- `wakeGracePeriod` is `SleepSchedule.wakeGracePeriod = 5 * 60`, commented as a chosen balance number.
- AC10 is satisfied INSIDE `wakeIfAsleep` (`guard ..., state.healthStatus != .dead`) rather than at
  each of the three call sites, so it is one rule rather than three copies. Bonus: the old code
  charged the waking-early mistake for a DEAD Digimon, which was a mistake for a disturbance that
  could not have happened. It no longer does.
- Decision worth recording: a woken Digimon that turns out to be SICK or BROKE is still blocked, and
  is still charged the disturbance. It really was disturbed — it is awake and walking about either
  way — and this is the same reasoning US-108 already applied to a battle with no energy.
