TWO REAL BUGS FOUND — later iterations should know both:
1. **WKWatchOnly: true was missing from Info.plist.** WKApplication alone BUILDS FINE but the
   app FAILS TO INSTALL: "This app does not indicate that it is Watch-only ... must specify
   WKCompanionAppBundleIdentifier". US-001 marked the plist verified, but it only ever ran
   `build` — install is what exposes this. Fixed in project.yml. Any story that runs tests or
   the Simulator depends on this key; do not remove it.
2. **`Bundle.url(forResource:)` returns an ARBITRARY png for an empty name.** It treats "" like
   nil, so `loadSheet(stage:"Child", name:"")` returned Morphomon, and stage "" returned
   "Pendulum Cycle.png" — a blank spriteFile would have silently loaded the WRONG Digimon
   instead of failing. SpriteLoader now guards `!name.isEmpty && !stage.isEmpty`. The US-009
   graph validator should treat an empty spriteFile as an error, not rely on a nil load.

Notes for next iterations:

  "Black and White Sprites/" (v2 at best, per PRD Non-Goals), "LCD Checklist.xlsx" (36K), and
  "Visual Checklist.png" (404K). `type: folder` cannot exclude children. Not in any AC, so left
