NEW DEBUG FLAG, and why it was needed
`-complicationEnergyDemo` makes `ComplicationDemoView` draw
`ComplicationSnapshot.longestEnergyLabel` (the placeholder wearing SLEEP, the
widest of the four). Without it the gauge label cannot be photographed at all:
the Simulator has no HealthKit data, so every game it publishes has NO dominant
energy and the face correctly draws "No energy yet". Hand-writing a snapshot
into the app-group JSON does not work either — the app republishes on launch and
overwrites it (verified: the file came back with `dominantEnergySymbol` absent).
The flag chooses only WHO is on the face; the views drawn are the shipping ones.

DESIGN NOTES FOR LATER STORIES
- `beats` is the SINGLE source of truth for the chart. The "therefore weak to"
  column in D-2 is never authored anywhere in code — the test derives it by
  reading `beats` backwards, so the two halves of the published table cannot
  drift apart.
- `effectiveness(against:)` answers "am I strong here?", not "who wins?".
  Advantage wins the tie, so light vs dark reports `.advantage` for BOTH sides
  (pinned). US-092 must do the cancelling arithmetic (1.25 x 0.8 = 1.0) itself;
  do not "fix" the mutual pair here.
- `neutral` and `free` are inert BECAUSE they are the US-087 lookup floor. If a
  later story gives either one a matchup, ~930 unauthored roster Digimon all
  silently inherit that advantage.
- Raw values are the wire format of US-087's `elements.json`. A test pins each
  one; renaming a case re-types every Digimon authored under the old name.

DESIGN NOTES FOR LATER STORIES
- ORDER IS THE COLLISION RESOLVER in `keywordRules`. Five rules exist only to be
  reached first: `trice` before `ice` (Triceramon is not an ice type),
  `cherubimon` before `ice` (`_vice`), `orange` before `ange` (Growmon Orange is
  not an angel), `aquila` before `aqua`, and `yuki`/`kaguya`/`raguel`/`haguru`
  before `agu`. Adding a rule near the top can silently re-type ids below it —
  re-run the whole-roster test and eyeball the per-keyword hits.
- Six rules the story REQUIRES currently match no roster id at all (`angel`,
  `machine`, `raidra`, `frigi`, `aqua`, `flame`). They are kept deliberately;
  the test asserts them through the lookup with a synthetic `zz<keyword>zz` id,
  so a rule that exists but is shadowed by an earlier one still fails.
- An explicit `types` entry beats the keyword table. `darkdramon` is the pinned
  case: a Vaccine machine dragon whose name says `dark`.
- Keyword-tier attributes are a genre heuristic, not canon: only the angel
  families (vaccine) and the devil/undead families (virus) are typed; every
  other rule says `free` rather than fabricating an attribute for a family.
- Elements agree with the Digimon's authored attack in `moves.json` — Meramon
  throws flame, so it is fire. US-088's badges will sit next to US-089's move
  row, so a mismatch between the two files would be visible on one screen.
